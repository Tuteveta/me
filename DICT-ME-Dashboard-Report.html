4. Module Functions

4.1 Annual Workplan  /workplan

Divisions create annual plans with Key Results Areas (KRAs) and KPIs for each fiscal year. Approved workplans become the source-of-truth for funding requests and quarterly report auto-population.

Core Creation

  • Set fiscal year, division, objective, and total budget (PGK)
  • Link to a Corporate Strategic Priority — KRAs auto-fill from the priority objectives
  • Add/edit KRAs with title, description, and weight (%)
  • Per KRA: add KPIs with name, unit, baseline, Q1–Q4 targets, annual target, responsible officer, measurement method

Status Workflow & Controls

  • Draft → Submitted → Approved → Active
  • Submit for management approval
  • Executive/Deputy/DCS approves or activates
  • Approved workplans link to Funding Requests and Quarterly Reports

4.2 Quarterly Reports  /quarterly-report

Branches submit quarterly performance reports against KRAs and activities. Executive Managers review and add timestamped comments.

Branch Manager Functions

  • Create Report — Set Wing, Division, Branch, Fiscal Year; select quarters (Q1–Q4)
  • Auto-populate from Workplan — Link approved workplan; entries generated from KRAs with KPI targets pre-filled
  • Edit entries — 11 columns: Quarter · KRA · Program/Project · Description of Key Activities · KPIs · Expected Outcomes · Approved Budget (K) · Quarter Expenditure (K) · Status · Justification/Remarks · Officers in Charge
  • Status options per entry: Completed · Ongoing · Not Started · Deferred
  • Add/delete rows · Submit report · Print with DICT letterhead · Export CSV

Executive / Management Functions

  • Review queue with count badge on sidebar and dashboard
  • Mark as Reviewed — add timestamped management comment (read-only to submitting officer)
  • Delete report with confirmation
StatusMeaningWho Can Act
DraftBeing prepared, not yet submittedBranch Manager (edit/delete)
SubmittedAwaiting reviewExecutive Manager (review)
ReviewedReviewed with commentRead-only for all

4.3 Budget Plan  /budget-plan

Branches plan and track budget line items — estimated costs vs actual utilization — submitted for management review each quarter. Plans support eight expenditure categories with auto-calculated utilization rates and variance summaries.

Branch Manager Functions

  • Create Plan — Set Wing, Division, Branch, Fiscal Year, and Plan Title
  • Add line items — Category, Description, Estimated Cost (K), Utilized (K), Justification; utilization % auto-calculated per item
  • 8 budget categories: Personnel · Goods & Services · Capital · Training · Travel · ICT · Consultancy · Operational
  • Summary totals — Total Estimated (K), Total Utilized (K), Variance (Surplus/Deficit), Overall Utilization %
  • Add/edit/delete line items (Draft state only)
  • Submit — advances plan to Submitted; locked for editing
  • Print — formatted plan with DICT letterhead
  • Export CSV — all line items (Category, Description, Estimated Cost, Utilized, Justification)
  • Delete plan (Draft state only, with confirmation)

Executive / Management Functions

  • Review all submitted budget line items with per-item utilization progress bars
  • Add Management Review comment — visible to Branch Manager after review
  • Mark as Reviewed — timestamped with reviewer name and date
  • Review queue count badge on sidebar and dashboard widget

Finance Manager Functions

  • View all budget plans across all Wings and Divisions (read-only)
  • Filter plans by Division, Fiscal Year, and Status
  • Cross-division totals — aggregate Estimated vs Utilized by category
  • Access via quick link on Finance dashboard widget

List View (All Roles)

  • Status filter cards: All · Draft · Submitted · Reviewed
  • Search by plan title or branch name
  • Filter by Division and Fiscal Year
  • Table columns: Title, Division, Fiscal Year, Total Estimated (K), Total Utilized (K), Utilization %, Status, Submitted By, Date
StatusMeaningWho Can Act
DraftBeing prepared, not yet submittedBranch Manager (edit items / delete plan)
SubmittedAwaiting management reviewExecutive Manager (review + comment)
ReviewedReviewed with management commentRead-only for all

4.4 Funding & Request Workflow  /requests  /approvals  /finance

Manages the full lifecycle of all staff requests through a defined role-based approval chain. Six request types with different routing logic.

Request TypeApproval ChainFunding Required
Funding RequestEM → Deputy → Director → Finance → AcquittalYes
Procurement RequestEM → Deputy → Director → Finance → AcquittalYes
Training RequestEM → DeputyNo
Policy / ComplianceEM → DirectorNo
Leave / TravelEM onlyNo
IT SupportEM onlyNo

Officer / Manager (My Requests)

  • Submit new request — select type, fill details, attach documents
  • Link funding requests to a specific Workplan and KRA
  • Visual approval chain tracker showing each approver's decision
  • Submit acquittal report (notes + attachments) after Finance approval
  • Track stage in real time

Approvers (EM / Deputy / DCS)

  • Approve — advance to next stage with comment
  • Reject — terminate with reason
  • Defer / Hold — pause pending more information (can un-defer)
  • View full prior approval chain history

Finance Manager (Finance Page)

  • Finance-stage approvals — review and approve/reject requests that completed the management chain
  • Assign DICT budget line code to approved requests
  • Acquittal review — accept or request revision on submitted acquittal reports
  • Close request after acquittal accepted — final record keeping

4.5 Expenditure Budget  /budget

View Functions (All roles)

  • Total Budget, Expended, Remaining, Utilisation % — summary cards
  • Quarterly filter (Q1 / Q2 / Q3 / Q4 / All)
  • Quarterly breakdown panel with % of quarterly budget
  • Accordion per functional area — expand/collapse
  • Per-program: Executive Manager, spent vs allocated, % expended
  • Per-line table: Allocated, Q1–Q4, Total Spent, Remaining, % bar
  • Grand total footer: Allocated, Expended, Remaining, Q3 YTD

Finance Manager — Edit Mode

  • Edit Data button activates inline editing (Finance + Super only)
  • Q1–Q4 cells become number inputs per budget line
  • Allocated budget editable per line and per area total
  • All totals, %, progress bars recalculate live
  • Save Changes — persists to localStorage
  • Cancel — discards all unsaved edits

4.7 Reports Registry  /reports

Unified registry aggregating all Quarterly Reports and Budget Plans. Status cards (Reviewed / Submitted / Draft) filter the table. Type filter for Quarterly vs Budget Plan. Shows: Title, Type (icon-coded), Division, Fiscal Year, Submitted By, Date, Status. Export CSV of filtered list.

4.8 KPI Monitoring  /kpi

Tracks six DICT programme KPIs with 6-month trend history and target comparison. Each KPI card shows: program area, unit, target vs actual, baseline, status badge (On Track / At Risk / Off Track / Exceeded), trend indicator, and last updated date. Filter by status. Expandable trend chart per KPI.

KPIs monitored: Government Cloud Adoption Rate (%) · Cybersecurity Incidents Resolved · ICT Policy Documents Reviewed · Partnership MOUs Signed · DevOps Pipeline Deployments · Staff ICT Training Completion

4.9 Projects  /projects

Tracks capital and programme projects. Each project card shows: name, program area, status badge, completion %, lead officer, budget vs spent, beneficiary count, and milestone tracker with due dates and completion ticks. Five active projects across all functional areas.

4.10 Corporate Plan  /corporate-plan

Displays DICT strategic priorities with objectives. Annual Workplans can be linked to a strategic priority so KRAs auto-populate from the priority's objectives — ensuring all divisional plans align to the Corporate Plan.

4.11 Organisation Chart  /organisation

Visual organisational structure: 4 Functional Areas → Programs → Activities/Branches. Shows head/role for each area. Colour-coded by wing. Lists Executive Managers per program and Branch Managers per activity.

4.12 My Team & Task Management  /my-team

Officer Management

  • Create officers: name, email, position, division, program, status
  • View full roster with active/inactive status
  • Deactivate/reactivate officers
  • Overview stats: total officers, active count

Task Assignment

  • Create tasks: title, description, priority, due date, program
  • Priority: Low / Medium / High / Urgent
  • Status: Pending / In Progress / Completed / Overdue
  • Edit progress (0–100%), add notes, mark complete
  • Filter by status · Task completion rate card

4.13 My Tasks (Officers)  /my-tasks

Officers view tasks assigned by their manager. Update progress %, add update notes, view priority and due date. Overdue tasks are automatically flagged based on due date. Filter by status and priority.

4.14 User Management  /users  Super Only

System-wide account management: view all users (name, email, role, division, status, last login, created date) · Create new accounts for any role and division · Edit role, division, status · Deactivate/reactivate accounts · Role distribution summary.

4.15 Help & User Guide  /help

Role-aware step-by-step guides that adapt to who is logged in. Three guide sets: Branch Manager (Budget Plan, Quarterly Report, Requests, Workplan) · Executive/Management (Review reports, Review budgets, M&E Summary, Approvals) · Finance Manager (Finance approvals, Acquittals, Budget tracking). System information panel with role, division, authentication, and ToR reference.

5. Approval Chain Workflows

Funding / Procurement Request

Submit Request → [Pending EM] → Approve → [Pending Deputy] → Approve → [Pending DCS] → Approve → [Pending Finance]
→ Approve + Budget Line → [Pending Acquittal]
→ Submit Acquittal → [Acquittal Review] → Accept → [Closed]

At any stage: Reject → [Rejected]  |  Defer → [Deferred] (can be reactivated)

Training Request

Submit → [Pending EM] → Approve → [Pending Deputy] → Approve → [Closed]

Policy / Compliance Request

Submit → [Pending EM] → Approve → [Pending DCS] → Approve → [Closed]

Leave / Travel & IT Support Requests

Submit → [Pending EM] → Approve → [Closed]

Quarterly Report Review Cycle

Branch Manager creates report → [Draft] → Submit → [Submitted]
→ Executive Manager reviews + comment → [Reviewed]

Budget Plan Review Cycle

Branch Manager creates plan → [Draft] → Submit → [Submitted] → Management reviews + comment → [Reviewed]
